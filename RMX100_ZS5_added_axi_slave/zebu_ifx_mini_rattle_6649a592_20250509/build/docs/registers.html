MNSTATUS

Name address size access description
MNSTATUS 0x744 32 read-write MNSTATUS Register. On entry to an RNMI trap handler the mnstatus CSR is updated with information from the interrupted context:

The mnstatus.IP bit shadows the rnmi_a input signal and is read-only. Writes to mnstatus do not affect the value of the IP bit.

The mnstatus.NMIE bit determines whether interrupts are enabled or disabled. On reset (including soft reset), the NMIE bit is set to 0, and in this state all interrupts are disabled - including RNMIs. When software is ready to accept RNMIs it sets NMIE to 1. This enables RNMIs and returns normal global control of maskable interrupts to mstatus.MIE.

Software cannot clear the NMIE bit - once it has been set it remains set until an RNMI is taken, when the NMIE bit is cleared.

The NMIE bit does not affect the operation of the WFI instruction.

In order to minimize latency until the next RNMI is taken, software should follow the top-half/bottom-half model, where the RNMI handler itself only en-queues a task to handle the interrupt and then returns (or sets the NMIE bit to 1). The handler should acknowledge the source of the interrupt and wait until it detects mnstatus.IP is clear before returning. The bulk of the interrupt servicing is performed later, with RNMIs enabled.

All other bits in mnstatus are reserved; software should write only zeros to those bits, and it will always return zeros when read by software.

MNPP is set to the privilege mode prior to taking the RNMI trap. The encoding of this field is equivalent to the mstatus.MPP field.

MNPV is set to the virtualization mode prior to taking the RNMI trap. The encoding of this bit is equivalent to the mstatus.MPV bit.

NMIE is cleared.
